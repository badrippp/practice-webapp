# Производственная практика — Веб-приложение "Hello World"

## Название проекта
Развёртывание веб-приложения с использованием Docker Compose и GitLab.

## Краткое описание
Простое статическое веб-приложение (HTML-страница), упакованное в Docker-образ на основе Nginx
и запускаемое с помощью Docker Compose. Исходный код хранится в репозитории GitLab, история
изменений содержит не менее трёх фиксаций.

Страница отображает:
- заголовок "Hello World";
- название практики;
- ФИО студента;
- номер группы.

## Используемые технологии
- Ubuntu Server 24.04 LTS
- Docker Engine
- Docker Compose
- Portainer / Dockhand — управление контейнерами
- GitLab Community Edition — хранение исходного кода
- Nginx — веб-сервер внутри контейнера

## Инструкция по запуску

1. Установить Docker и Docker Compose на сервере (Ubuntu Server 24.04 LTS).
2. Склонировать репозиторий проекта:
   ```bash
   git clone <URL-репозитория-в-GitLab>
   cd <папка-проекта>
   ```
3. Собрать образ и запустить контейнер одной командой:
   ```bash
   docker compose up -d
   ```
4. Проверить, что контейнер запущен:
   ```bash
   docker ps
   ```
5. Проверить работу приложения:
   ```bash
   curl http://IP-адрес-сервера
   curl -i http://IP-адрес-сервера
   ```
   Ожидаемый код ответа: `HTTP/1.1 200 OK`.

## Описание используемых контейнеров

| Контейнер         | Образ                | Назначение                                 | Порт  |
|-------------------|----------------------|---------------------------------------------|-------|
| practice_webapp   | practice-webapp:latest (собран из `nginx:alpine`) | Отдаёт статическую HTML-страницу | 80:80 |
| gitlab            | gitlab/gitlab-ce     | Хранение исходного кода, история изменений  | 80/443/22 (см. отдельный запуск) |

> Примечание: GitLab CE запускается отдельным контейнером (см. раздел "Развёртывание GitLab" в отчёте) и не включён в `docker-compose.yml` веб-приложения, так как используется как отдельная инфраструктурная система.

- Исходный код размещён в репозитории GitLab, выполнено не менее трёх фиксаций изменений.
- Проверка через `curl` подтвердила корректную работу приложения (код ответа 200 OK).
